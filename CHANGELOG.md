# Changelog

Все заметные изменения проекта документируются здесь. Каждый release вытягивает
содержимое своего раздела (между двумя `## vX.Y.Z` заголовками) в описание
GitHub Release автоматически — через workflow `.github/workflows/build.yml`.

## v0.9.5

### Новые возможности

- **Авто-фильтр «только Европа»**. Новые подписки сразу включают в авто-режим
  только европейские серверы, а всё с «Обход»/«Bypass» в имени исключают.
  Кнопка «Авто-фильтр (Европа)» применяет это правило к уже добавленным
  профилям задним числом.
- **Колонка «В авто»** в списке профилей (✓/✗). Один клик по ячейке
  включает/выключает участие сервера в авто-режиме — выделение при этом не
  сбрасывается.
- **Внешний IP после подключения**. Под активным сервером появляется реальный
  внешний IP через туннель — мгновенное подтверждение, что VPN работает.
- **Меню в трее с выбором сервера**. ПКМ по иконке → «Подключить к…» со
  списком серверов: можно переключиться, не открывая окно.
- **Поиск/фильтр профилей**. Поле над списком фильтрует по имени или адресу
  сервера.
- **«Закрепить в трее»** в меню-бургере открывает «Параметры → Панель задач»,
  чтобы вынести иконку MeDVeD из скрытых значков. Плюс стабильный
  AppUserModelID — Windows теперь относится к иконке как к полноценному
  приложению.
- **Автозапуск через Планировщик задач** вместо ключа реестра `Run`. Ключ
  `Run` стартует приложение без прав администратора, поэтому автоподключение
  при загрузке упиралось в запрос UAC. Теперь задача с «наивысшими правами»
  запускает MeDVeD сразу с правами — автоподключение на старте работает без
  единого окна UAC. Настройка автозапуска требует разового запуска от имени
  администратора. Если приложение свёрнуто в трей и прав нет — автоподключение
  не зависает на невидимом диалоге, а пишет в журнал и пропускается.

### Под капотом — надёжность

- **Persistent log**. Все записи журнала теперь пишутся в
  `%LOCALAPPDATA%\vless_manager\app.log` с ротацией по 1 МБ. Лог переживает
  крах и помогает диагностировать что произошло.
- **Глобальный обработчик исключений**. `sys.excepthook` и
  `threading.excepthook` пишут traceback'и в `app.log`. Frozen-exe больше не
  умирает молча.
- **Preflight `sing-box check`** перед запуском. Если конфиг невалиден,
  пользователь увидит понятное сообщение, а не загадочный «VPN упал код 1».
- **Self-heal**. Если sing-box упал не по команде пользователя и проработал
  ≥10 секунд — одна автоматическая попытка переподключения через 3 секунды.
  Защита от бесконечного зацикливания на постоянных ошибках.
- **Очистка TUN-адаптеров**. При старте отключаем «зависшие» `singtun*`
  адаптеры от предыдущих крахов. Решает «адаптер уже существует».
- **Снятие «осиротевшего» kill switch**. Правило фаервола от kill switch
  переживает перезагрузку. Если приложение умерло жёстко (Диспетчер задач,
  краш, выключение питания) с активным kill switch, интернет оставался
  заблокированным без всякой подсказки. Теперь при старте такое стухшее
  правило снимается автоматически.
- **Исправлен краш трея при наличии профилей**. Пункты подменю «Подключить
  к…» создавались лямбдой с тремя аргументами, а pystray отвергает callback
  с более чем двумя — из-за чего иконка трея вообще не запускалась, как
  только в списке появлялся хотя бы один сервер. Теперь сервер захватывается
  через замыкание (два аргумента) — трей работает.
- **Крестик сворачивает в трей, а не закрывает**. Приложение продолжает
  работать в фоне; полностью выйти можно через «✕ Выход» в меню или в трее.
  Если трей создать не удалось — крестик честно закрывает приложение, а не
  прячет его невидимым процессом.
- **Системные уведомления Windows отключены полностью**. Раньше падение VPN,
  активация kill switch и т.п. всплывали тостом в углу экрана. Теперь такие
  события только пишутся в журнал и (если окно открыто) показываются внутри
  приложения — никаких всплывашек на рабочем столе. Зависимость `plyer`
  убрана, сборка стала чуть легче.

### Под капотом — CI

- **GitHub Actions**. Push тега вида `vX.Y.Z` → CI скачивает sing-box,
  собирает MeDVeD.exe через PyInstaller и прикрепляет к Release. Тело
  релиза подтягивается из этого файла, секция по тегу.

### UX

- **Шрифты на главной**. «Сервер:», «Активный сервер:», статистика и
  информация о подписке стали крупнее и читаемее.
- **Информация о подписке**. Если провайдер не отдаёт `subscription-userinfo`,
  пользователь видит явное «(подписка не сообщает лимиты)», а не пустоту.
  Заголовок логируется в журнал — видно факт получения/отсутствия.
- **Пинг = реальный TLS-handshake**, усреднённый по нескольким попыткам и
  параллельный (до 8 потоков). Это тот же показатель, что в Karing —
  реалистичные 50–200 мс вместо обманчивых `<1 мс` от голого TCP-пинга.
  Sub-10мс показывается с десятичной долей (`3.4 мс`).
- **Минимальный размер окна 1000×720** — правая панель «Импорт ссылки /
  Подписка» больше не обрезается на дефолтном размере.
- **Копирование ссылки чинит экспорт не-VLESS профилей**. Раньше кнопка для
  trojan/ss/hysteria2/tuic выдавала сломанный `vless://` с пустым UUID —
  теперь собирает корректную ссылку под нужный протокол.
- **Поле «Путь к sing-box»** убрано из настроек — бинарник встроен,
  настройка только сбивала с толку. Power-user может задать кастомный путь
  вручную через `settings.json`.
- **Настройки сохраняются сразу** при любом изменении — кнопка «Сохранить»
  больше не нужна.
