# Changelog

Все заметные изменения проекта документируются здесь. Каждый release вытягивает
содержимое своего раздела (между двумя `## vX.Y.Z` заголовками) в описание
GitHub Release автоматически — через workflow `.github/workflows/build.yml`.

## v0.11.1

### Надёжность

- **sing-box больше не остаётся «осиротевшим».** Процесс sing-box теперь
  привязан к Job-объекту Windows с автоубийством: если приложение завершилось
  жёстко (краш, Диспетчер задач, выключение питания), Windows гарантированно
  закроет и sing-box вместе с ним. Раньше в таком случае туннель мог остаться
  поднятым и перехватывать маршруты/DNS — без интернета и с аномально низким
  пингом — до следующего запуска. Обычные «Отключить»/«Выход» и так корректно
  останавливали VPN; это закрывает именно аварийные сценарии.

## v0.11.0

### Обход блокировок (DPI / ТСПУ)

- **Фрагментация TLS — включена по умолчанию.** Разбивает TLS-рукопожатие так,
  что DPI (в том числе российский ТСПУ) не видит имя сайта (SNI) и не может
  заблокировать соединение по нему. Серверам с Reality не применяется — им это
  не нужно. Если на каком-то сервере мешает — отключается в Настройках.
- **Агрессивный режим фрагментации.** Переключатель для упрямого DPI, когда
  обычной фрагментации не хватает: дробит на уровне пакетов (чуть медленнее).
- **Маскировка TLS (uTLS).** Выбор отпечатка TLS-клиента (chrome, firefox,
  safari, randomized и т.д.), чтобы уходить от фингерпринтинга прокси-клиента.
  По умолчанию «auto» — берётся из ссылки сервера.
- **Шифрованный DNS (DoH) — опционально.** Резолвинг через зашифрованный DNS
  поверх туннеля, чтобы провайдер не мог подменять DNS-ответы (одна из техник
  ТСПУ). По умолчанию выключено, так как меняет путь резолвинга; включается в
  Настройках. Адреса серверов при этом резолвятся напрямую, чтобы не было
  «петли» при подключении.

Все эти настройки — в Настройки → «Обход блокировок (DPI / ТСПУ)». Применяются
после переподключения.

## v0.10.0

### Профили и подписки

- **Флаги стран в списке серверов.** Перед именем рисуется настоящий флаг.
  Раньше Windows показывал флаг-эмодзи из имени как «de/fl» — в его шрифте
  попросту нет глифов флагов. Теперь флаги рисуются программно (без картинок в
  комплекте), а сломанный эмодзи из текста убирается. Для стран, которых нет в
  наборе, остаётся чистое имя без мусора.
- **Имя подписки вместо ссылки.** Если провайдер присылает название (заголовок
  `Profile-Title` или `Content-Disposition`), оно показывается вместо длинного
  URL — и в шапке группы профилей, и в новой панели подписок. Нет имени —
  остаётся URL, как раньше.
- **Панель «Мои подписки».** Список подписок, у каждой свои кнопки: ⟳ обновить
  только эту подписку и ✕ удалить её вместе с серверами. Рядом — остаток
  трафика и срок по этой подписке, если провайдер их сообщает.
- **Информационные строки больше не теряются.** Серверы-заглушки и инфо-строки
  (`127.0.0.1:1`, `localhost`, «Приложение не поддерживается») раньше молча
  выкидывались. Теперь они видны серым как «ℹ️ … / инфо», но остаются
  инертными: не участвуют в авто-режиме, к ним нельзя подключиться, и их нет в
  меню выбора сервера и в трее.
- **Выбор процесса из списка.** В правиле по процессам (Настройки) можно выбрать
  запущенный процесс из фильтруемого списка, а не вводить имя `.exe` вручную.

### Пинг

- **Пинг к серверам без отключения VPN.** Теперь, не разрывая подключение, можно
  замерить реальную задержку до каждого сервера — измерение идёт через сам
  sing-box (его Clash API), как это делает Karing. При отключённом VPN работает
  прежний прямой замер.

### Под капотом

- **Тесты.** Набор автотестов расширен с 29 до 173 (парсер всех протоколов,
  сборка конфига, модели, storage, утилиты обновления, флаги). Покрытие
  ключевой логики проверяется в CI с порогом — регрессии ловятся до релиза.

## v0.9.9

### Надёжность

- **Контрольная сумма SHA-256.** К каждому релизу теперь прикладывается
  `MeDVeD-Setup-X.Y.Z.exe.sha256`, а авто-обновление сверяет хэш скачанного
  установщика перед запуском — защита от повреждённой или подменённой загрузки
  (в дополнение к проверке размера). Если сумма не совпала, установка
  отменяется и приложение не закрывается.
- **Тесты и CI-проверка.** Добавлен набор тестов (pytest): round-trip всех
  протоколов в парсере, сборка конфига sing-box (включая критичные DNS-правила),
  (де)сериализация настроек. Отдельный workflow гоняет их на каждый push/PR —
  регрессии ловятся до релиза.
- **Версия sing-box запинена.** CI собирает с конкретной версией sing-box
  (1.13.12) и проверяет, что бинарь рапортует именно её, — сборки стали
  воспроизводимыми, а ломающее обновление sing-box не утечёт в релиз незаметно.

### UX

- **Прогресс-бар при скачивании обновления** — вместо «зависшего» окна виден
  процент и мегабайты.
- **«Что нового» после обновления.** После авто-апдейта один раз показывается
  список изменений новой версии (из CHANGELOG, с оформлением — заголовки,
  списки, выделения).
- **Кнопка «Скопировать диагностику»** в Журнале — копирует версии, окружение и
  хвост лога для багрепортов (без ссылок-подписок и паролей).
- **Анимация переключения экранов** — разделы из меню-бургера въезжают плавным
  слайдом (вглубь — справа, назад — слева).

### Пинг

- **Пинг в списке профилей теперь 1-RTT (TCP)** — тот же порядок чисел, что
  показывает Karing, вместо вдвое больших значений от полного TLS-рукопожатия.
  Точную задержку через тоннель для активного сервера по-прежнему показывает
  поле «Пинг:» (через Clash API самого sing-box).

## v0.9.8

### Установка и настройки

- **Выбор папки установки.** Установщик снова показывает страницу «Куда
  установить» — можно поставить MeDVeD в любую папку, а не только в
  предложенную по умолчанию (`%LOCALAPPDATA%\Programs\MeDVeD`). При тихом
  авто-обновлении путь не спрашивается — берётся тот, что выбран при установке.
- **Авто-обновление подписки по умолчанию — раз в час.** Раньше у новых
  установок интервал стоял `0` (выключено); теперь по умолчанию `1` час.
  Выключить, поставив `0`, по-прежнему можно — это значение больше не
  сбрасывается при перезапуске.

## v0.9.7

### Главное — переход на установщик

- **Новый формат поставки: установщик `MeDVeD-Setup-X.Y.Z.exe`** вместо одного
  «портативного» exe. Это убирает ошибку первого запуска
  `Failed to load Python DLL ... LoadLibrary: Не найден указанный модуль`,
  которая могла выскочить сразу после обновления.
  - **Почему она возникала.** Старый «один файл» при каждом запуске
    распаковывал ~1000 вложенных файлов (включая `python313.dll`) во временную
    папку `_MEI...` и тут же их загружал. На самом первом запуске только что
    записанного файла эта распаковка конкурировала с антивирусом, который в тот
    же момент сканировал новый exe, — и загрузка `python313.dll` падала. При
    повторном запуске всё работало.
  - **Как исправлено.** Теперь приложение ставится в папку
    `%LOCALAPPDATA%\Programs\MeDVeD` и при запуске **ничего никуда не
    распаковывает** — `python313.dll` лежит рядом и грузится напрямую. Ошибка
    `_MEI` стала физически невозможной.
- **Установка без UAC.** Установщик ставит приложение «для текущего
  пользователя», без запроса прав администратора. Создаёт ярлык в меню «Пуск»
  (и, по желанию, на рабочем столе) и пункт удаления в списке программ.
- **Тихое авто-обновление.** «Обновить» теперь скачивает установщик и запускает
  его в тихом режиме: он сам дожидается закрытия старой версии, заменяет файлы и
  перезапускает MeDVeD — без окон и без ошибки первого запуска.

### Замечание про первый запуск

- Остаётся только штатное предупреждение Windows SmartScreen «Неизвестный
  издатель» при первом запуске установщика (один раз: «Подробнее → Выполнить
  в любом случае»). Оно есть у любого неподписанного приложения; полностью его
  убирает только подпись сертификатом (платно).

### Под капотом — сборка

- PyInstaller переведён с `--onefile` на `--onedir` (папка `MeDVeD\` с
  `MeDVeD.exe` и `_internal\`). Отключён UPX (он тоже провоцировал ложные
  срабатывания антивируса). CI собирает папку и упаковывает её установщиком
  Inno Setup (`MeDVeD.iss`), который и прикрепляется к Release.

## v0.9.6

### Новые возможности

- **Авто-фильтр «только Европа»**. Новые подписки сразу включают в авто-режим
  только европейские серверы, а всё с «Обход»/«Bypass» в имени исключают.
  Кнопка «Авто-фильтр (Европа)» применяет это правило к уже добавленным
  профилям задним числом.
- **Колонка «В авто»** в списке профилей (✓/✗). Один клик по ячейке
  включает/выключает участие сервера в авто-режиме — выделение при этом не
  сбрасывается.
- **Внешний IP после подключения**. Под активным сервером появляется реальный
  внешний IP через туннель — мгновенное подтверждение, что VPN работает.
- **Меню в трее с выбором сервера**. ПКМ по иконке → «Подключить к…» со
  списком серверов: можно переключиться, не открывая окно.
- **Поиск/фильтр профилей**. Поле над списком фильтрует по имени или адресу
  сервера.
- **«Закрепить в трее»** в меню-бургере открывает «Параметры → Панель задач»,
  чтобы вынести иконку MeDVeD из скрытых значков. Плюс стабильный
  AppUserModelID — Windows теперь относится к иконке как к полноценному
  приложению.
- **Автозапуск через Планировщик задач** вместо ключа реестра `Run`. Ключ
  `Run` стартует приложение без прав администратора, поэтому автоподключение
  при загрузке упиралось в запрос UAC. Теперь задача с «наивысшими правами»
  запускает MeDVeD сразу с правами — автоподключение на старте работает без
  единого окна UAC. Настройка автозапуска требует разового запуска от имени
  администратора. Если приложение свёрнуто в трей и прав нет — автоподключение
  не зависает на невидимом диалоге, а пишет в журнал и пропускается.

### Под капотом — надёжность

- **Persistent log**. Все записи журнала теперь пишутся в
  `%LOCALAPPDATA%\vless_manager\app.log` с ротацией по 1 МБ. Лог переживает
  крах и помогает диагностировать что произошло.
- **Глобальный обработчик исключений**. `sys.excepthook` и
  `threading.excepthook` пишут traceback'и в `app.log`. Frozen-exe больше не
  умирает молча.
- **Preflight `sing-box check`** перед запуском. Если конфиг невалиден,
  пользователь увидит понятное сообщение, а не загадочный «VPN упал код 1».
- **Self-heal**. Если sing-box упал не по команде пользователя и проработал
  ≥10 секунд — одна автоматическая попытка переподключения через 3 секунды.
  Защита от бесконечного зацикливания на постоянных ошибках.
- **Очистка TUN-адаптеров**. При старте отключаем «зависшие» `singtun*`
  адаптеры от предыдущих крахов. Решает «адаптер уже существует».
- **Снятие «осиротевшего» kill switch**. Правило фаервола от kill switch
  переживает перезагрузку. Если приложение умерло жёстко (Диспетчер задач,
  краш, выключение питания) с активным kill switch, интернет оставался
  заблокированным без всякой подсказки. Теперь при старте такое стухшее
  правило снимается автоматически.
- **Исправлен краш трея при наличии профилей**. Пункты подменю «Подключить
  к…» создавались лямбдой с тремя аргументами, а pystray отвергает callback
  с более чем двумя — из-за чего иконка трея вообще не запускалась, как
  только в списке появлялся хотя бы один сервер. Теперь сервер захватывается
  через замыкание (два аргумента) — трей работает.
- **Крестик сворачивает в трей, а не закрывает**. Приложение продолжает
  работать в фоне; полностью выйти можно через «✕ Выход» в меню или в трее.
  Если трей создать не удалось — крестик честно закрывает приложение, а не
  прячет его невидимым процессом.
- **Системные уведомления Windows отключены полностью**. Раньше падение VPN,
  активация kill switch и т.п. всплывали тостом в углу экрана. Теперь такие
  события только пишутся в журнал и (если окно открыто) показываются внутри
  приложения — никаких всплывашек на рабочем столе. Зависимость `plyer`
  убрана, сборка стала чуть легче.

### Под капотом — CI

- **GitHub Actions**. Push тега вида `vX.Y.Z` → CI скачивает sing-box,
  собирает MeDVeD.exe через PyInstaller и прикрепляет к Release. Тело
  релиза подтягивается из этого файла, секция по тегу.

### UX

- **Шрифты на главной**. «Сервер:», «Активный сервер:», статистика и
  информация о подписке стали крупнее и читаемее.
- **Информация о подписке**. Если провайдер не отдаёт `subscription-userinfo`,
  пользователь видит явное «(подписка не сообщает лимиты)», а не пустоту.
  Заголовок логируется в журнал — видно факт получения/отсутствия.
- **Пинг = реальный TLS-handshake**, усреднённый по нескольким попыткам и
  параллельный (до 8 потоков). Это тот же показатель, что в Karing —
  реалистичные 50–200 мс вместо обманчивых `<1 мс` от голого TCP-пинга.
  Sub-10мс показывается с десятичной долей (`3.4 мс`).
- **Минимальный размер окна 1000×720** — правая панель «Импорт ссылки /
  Подписка» больше не обрезается на дефолтном размере.
- **Копирование ссылки чинит экспорт не-VLESS профилей**. Раньше кнопка для
  trojan/ss/hysteria2/tuic выдавала сломанный `vless://` с пустым UUID —
  теперь собирает корректную ссылку под нужный протокол.
- **Поле «Путь к sing-box»** убрано из настроек — бинарник встроен,
  настройка только сбивала с толку. Power-user может задать кастомный путь
  вручную через `settings.json`.
- **Настройки сохраняются сразу** при любом изменении — кнопка «Сохранить»
  больше не нужна.
